JQuery

jQuery 是第三方开发的执行Dom操作的极简化的函数库。

这是jQuery文档

jQuery 是第三方开发的执行Dom操作的极简化的函数库。 第三方：需要下载引入/ 执行DOM操作：jQuery还是在执行Dom操作 学习jQuery还是在学习dom 五件事情：增删改查 事件绑定 极简化：jQuery是对DOM中每步操作都进行了终极的简化 但是 做事情的步骤没有简化 jQuery中一切都是用函数来解决问题 没有属性。

为什么使用jQuery

  1. 简单
  2. 解决了大部分浏览器兼容问题 凡是jQuery让用的 都没有兼容性问题
  3. 今后pc端的项目 以及主流的框架底层 都是用jQuery实现的。
  4. 趋势:jQuery仅适用于PC端 不适用于移动端。

如何使用jQuery

  1. 官方网站下载：jquery.com
    1. 1.x 唯一支持旧浏览器的版本
    2. 2.x 不再支持旧浏览器
    3. 3.x 不再只是旧浏览器 增加新功能。

  $btn.click(function(){

      // var n=btnself.html().replace(/[^\d]*/g,'');

jQuery 函数的三大特点

  1. 自带遍历

  2. 凡是涉及修改的函数，都是一个函数两用

  3. 多数函数都会返回正在操作的jq对象主语 为了方便链式操作 下一个函数的调用可以直接接在上一个函数调用之后。前提:前一个函数返回的值必须是下一个函数所需的主语

    $('#btn').css('color','red').click(function(){...}).blur(function(){...})

  4. 凡是对一个元数据进行多个操作 尽量使用链式操作。

页面加载后自动执行

通常js代码（查找元素 事件绑定）都必须在页面内容加载完成后 才能执行。

  1. 将js代码放在<body>元素内容底部
  2. 通过窗口绑定 onload事件 事件会在所有网页内容（html,css,js,图片）都加载完以后 才自动触发 问题：最后一个会覆盖掉前边的onload 事件
  3. 使用事件监听：Window.addEventListener(‘load’,function(){…}) 支持多个。
$(window).load(function(){
	//....
}) //相当于on('load') / bind('load')

页面DOM准备就绪后 自动执行

$(document).ready(function(){
	//...
})  //网页dom完之后自动执行

// 简化写法
${function(){
  //...
}} //直接放在 大括号里。

有些功能和样式与图片无关 就不需要放入load里执行，应该dom就绪后就执行。

动画

  1. 简单动画:写死的固定 不变的三种动画效果

    1. 显示隐藏：.show()、.hide()、.toggle()

      问题：.show() .hide() 不加参数时,默认使用display:block 和 display:none控制瞬间的显示和隐藏 如果想要有动画效果 必须加持续时间参数。

    2. 上滑下滑：.slideUp()、.slideDown()、.slideToggle()

    3. 淡入淡出：.fadeIn()、.fadeOut()、.fadeToggle()

    简单的动画所有函数的共享问题：

    1. 效果是写死的 几乎不可维护。
    2. 用Js定时器和DOM操作的模拟动画效率非常低。

  2. 万能动画

    可以对任意css属性应用动画效果 主要希望对任意css属性应用动画效果时使用 只需要写目标值 可自动获得当前的对应属性值 自动计算变化的距离

    $('#d1').animate({css属性:目标值},持续时间) //只支持单个数组的css属性 不支持css3动画 不支持颜色过渡。

  3. 排队和并发

    1. 并发执行：多个css属性同时变化

    2. 排队执行：多个css属性排队变化

      $('#d1').animate({css属性:目标值},持续时间).animate({css属性:目标值},持续时间,function(){
  //动画完了以后执行的回调函数
}) //按顺序排队执行
      
//判断元素是否在运行动画
$('#d1').is(':animated') 
      
//停止动画 添加true参数停止所有 不加停止当前动画
$('#d1').stop([true])

    注:animate先加入到队列 然后顺序执行 不是立刻播放的意思

Ajax跨域

Access to XMLHttpRequest at 'http://127.0.0.1:3000/' from origin 'http://127.0.0.1:5000' has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource.
//ajax跨域请求会根据同源策略（Cross Origin Resources Shareing）被终止。

一个网站的网页中发送Ajax请求另一个网站的api:

    1. 可以请求成功 也可以获得数据 
    2.  浏览器会检查响应头(快递单)中的来源地址 如果来源地址和当前网页所在的地址不同则禁止使用该数据并报错

跨域请求包括：

  1. 域名不同: http://www.a.com ajax 请求 http://www.b.com的页面
  2. 子域名不同 http://www.a.com ajax 请求 http://bbs.a.com的页面
  3. 端口不同 http://www.a.com:80 ajax 请求 http://www.a.com:8888的页面
  4. 协议不同 http://www.a.com ajax 请求 https://www.a.com的页面

解决方法

  1. CORS 只需要服务端在返回数据时，在响应头中伪造来源地址，和客户响应头一致。Access-Control-Allow-Origin:'*' 任何地址都可以
  2. JSONP: 本地创建一个函数，让远端JS调用。

NodeJS 解决

npm i -save  cors //安装cors
app.js  //添加中间件函数
app.use(cors({
	origin:"http://127.0.0.1:5501"
}));

  2. 创建jq对象直接封装dom元素：$(this) 、$(e.target)
