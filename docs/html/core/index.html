HTML5

W3C组件 2014年推出的一组网页中技术总称

视频

  1. 视频文件格式 .mp4 .flv .webm .ogg

  2. 解码器：如果浏览器播放指定格式视频需要安装 此种格式视频对应解码器软件 浏览器安装了大部分视频格式解码器。

    标准语法

    1. controls 显示播放原生控件（兼容性查）
    2. loop 循环播放
    3. muted 静音播放
    4. preload 预加载策略
      1. none 不加载任何数据
      2. metadata 只预加载元数据
      3. auto 加载元数据且加载一部分时长的视频（默认）
    5. poster 在播放视频之前显示一张图片(广告) 如果视频暂停广告不再显示

    JS常见属性方法 必须通过js程序获取视频对象

    1. volume: 音量（0~1）
    2. playbackRate: 播放速率 大于1快放 小于1 慢放
    3. play() 播放视频
    4. pause() 暂停播放
    5. paused 表示当前视频播放状态 true表示暂停 false表示正在播放。

    video事件

    1. canplaythrough 当视频加载结束后可以播放时触发事件(一次)
    2. duration 视频时长(秒)
    3. ended 当视频播放结束后触发(一次)
    4. timeupdate ship 视频正在播放中(多次) 4/s
    5. currentTime 当前播放时间点。

    video与图片的专用样式 object-fit

    1. fill 拉伸 默认值
    2. contain 包含
    3. cover 覆盖
    <video src="文件路径">你的浏览器版本太低，请升级</video>
<!--属性-->

    兼容语法

    <video>
  <source src="x.mp4" />
  <source src="x.flv" />
  <source src="x.webm" />
</video>

    1. js操作svg画布

<script>
  // 获取画布
  var svg = document.getElementById("svg");
  // 创建圆环元素
  var c = document.createElementNS("http://www.w3.org/2000/svg", "circle");
  // 添加属性
  c.setAttribute("cx", 50);
  c.setAttribute("cy", 50);
  c.setAttribute("r", 50);
  c.setAttribute("fill", "red");
  // 添加到画布
  svg.appendChild(c);
  // 单击事件
  c.onclick = function() {
    // 获取半径
    r = this.getAttribute("r");
    this.setAttribute("r", (r *= 1.02));
  };
</script>

拖动和释放（HTML5）

  1. PC项目 拖动上传图片
  2. 移动端项目 自拍/相册

拖动源对象 触发三个事件

  1. dragstart 拖动开始
  2. drag 拖动中
  3. dragend 拖动结束

整个过程：dragstart +drag*n + dragend

拖动目标对象 触发四个事件

  1. dragenter 拖动进入

  2. dragover 拖动悬停 默认事件会影响拖动释放 需要先阻止默认行为

    event.preventDefault(); //阻止默认事件

  3. dragleave 拖动离开

  4. drop 拖动释放

Web Worker

程序：可以被CPU执行代码 程序存储在磁盘上

进程：将程序调用内存中并且分配指定空间，在内存中的程序称为进程

线程：进程内部是有多个线程组件==（内存）==

Chrome:一个Chrome浏览器进程内部至少有6个线程负责专项服务。

1. ***请求线程***： 发送请求 获取资源。
2. ***UI主线程***：负责绘制所有资源并且执行js 程序 

// 创建新进程执行js 不影响Ui绘制界面
var w=new Worker('js/05.js');

接收与数据

w.postMessage('发送给Worker');
w.onmessage=function(event){
  console.log('event',event.data)
}

注意事项：Worker中执行的代码不能包含DOM/BOM元素 操作DOM/BOM只能交给Ui主线程。 其他线程不能操作 以免造成混乱。

webSocket(重点)

  1. HTTP：超文本传输协议 用来传输网页中的资源（html/css/image/video…）请求/响应。一次请求对应一次响应，如果没有请求就没有响应。
  2. websocket：解决了数据实时响应问题。 工作方式 广播和收听

使用方法

  1. 服务器端： node.js / scoket.io 简单方便

    // 创建Web服务器
const app = require("http").createServer();
   
// 监听端口
app.listen(3001);
// 创建socket.io对象
const io = require("socket.io")(app);
   
// 接收客户端请求
io.on("connection", client => {
  console.log("有客户端连接到服务器");
  // socket 为当前客户创建专属服务。
  client.on("message", data => {
    console.log("data :", data);
  });
  var i=0;
  var t = setInterval(() => {
    client.emit("server", "发送给客户端"+(i++));
    if(i>5){
      clearInterval(t);
    }
  }, 500);
     
  //io 调用emit 表示 广播 所有客户端都可以接收到
  io.emit("news", "明天统一打八折");
     
  // 客户端离开事件。
  client.on("disconnect", data => {
    console.log(`客户端离开 :${data}`);
  });
});

webStroage

在客户端浏览器中保存用户专有的数据 保存：登录用户的昵称 购物车等。

客户端存储 服务器端存储
不安全 存储量小 海量存储。数据安全
cookie 兼容性好 缺点L操作复杂 数据库
flash 缺点 依赖flash播放器 session 对象
Webstorage 操作简单8MB 兼容性查 文件
indexDB 不标准 内存数据库 文档数据库

WebStorage 存储数据依赖两个对象

  1. sessionStorage 对象存储范围在当前会话中 当前会话中所有网页都可以使用 sessionStorage 但是如果会话结束后 数据丢失。
  2. LocalStorage 此对象保存数据永久保存

操作方法

  1. 保存数据 setitem(key,value) 键值对
  2. 获取数据 getitem(key) 通过键获取数值
  3. 删除指定数据 removeitem(key) 删除key键指定的数据
  4. 清空所有数据 clear() 清空
  5. 数据长度 length
// 存储数据
sessionStorage.setItem('name','马云老师');
localStorage.setItem('abc','我爱我的国家');

// 获取长度
console.log('length :', sessionStorage.length);
console.log('length :', localStorage.length);

// 获取列表
console.log('name :', sessionStorage.getItem('name'));
console.log('abc :', localStorage.getItem('abc'));
