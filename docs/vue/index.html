vue的基础学习笔记

什么是Vue

第三方开发的渐进式的基于MVVM设计模式的纯前端JS框架， 需要第三方下载。

  1. 渐进式：可逐步在项目中引入Vue的各项功能，且可与其他技术混用 今后极力不推荐混合使用。
  2. 利用MVVM设计模式：纯前端没有后端技术，Vue照常运行。
  3. Js框架：已具有核心功能的半成品项目代码。开发人员只需要根据具体业务的需求，添加定制自己个性化的那一部分内容。

所有以数据为主的项目，都可以使用Vue框架

如何使用Vue

两种下载方式

  1. 用<script>引入现有的vue.js到网页中 官方网址 初学者使用,学习单项Vue的技能。
  2. 安装脚手架代码

例子

<div id="app">
  <button @click="add">click me({{n}})</button>
</div>
<script src="js/vue.js"></script>
<script>
  var data = { n: 0 };
  var app = new Vue({
    el: "#app", //el要监控内容的父元素的选择器
    data,
    methods: {
      //交互事件函数写在这里
      add() {
        this.n++;
      }
    }
  });
</script>

MVVM

现代web前端开发重新划分了三步

  1. 视图/界面（view）:
    • HTML+CSS
    • 增强了HTML和CSS的功能，比如提供了变量、分支、循环等这些程序必备功能
  2. 模型数据（Model）:其实就是一个Data对象，统一保存这个界面所需的所有变量。
  3. 控制器（viewModel）：自动失效模型数据data中的内容和界面(view)中的数据实时同步。控制器封装了传统的增删改查功能，所以避免了一切的重复操作。

vue对象中包含两大子系统

  1. 响应系统

    实时监控data中的变量变化，自动为每个变量添加了访问器属性，这些访问器属性，直接列属于new Vue()对象 保护/监控data中的变量。而且所有访问器属性的set方法中内置了通知机制，只要视图修改data中的变量值，自然就会触发通知。

  2. 虚拟DOM树

    Vue内存中仅保存可能变化的Dom元素和可能变化的属性的简化版的dom树。仅包含可能变化的元素和可能变化的属性 遍历块并且已经封装了增删改查的操作。效率高 仅修改可能受影响的元素和可能受影响的属性，多数无关的元素和属性不受影响。

无论是Data中的变量还是methods中的函数 只要进入 new Vue()直接隶属于new Vue 可以直接使用 this.访问

双花括号

只要是 合法的有返回值的js 表达式都可以放入双花括号中，但是不支持 if else do/while for等循环体

不能用于绑定html片段 如果使用 会原样显示出来 不能在双花括号里修改变量 否则会造成死循环

指令

为HTML添加增强功能的特殊属性 传统的HTML缺少程序必须的要素,如：变量、函数、支、循环等 导致JS重复操作量巨大。

Vue一共有13个指令

  1. v-bind /: 属性绑定

    <!-- 属性绑定需要使用v-bind  val可以是合法的JS条件表达式 -->
<div v-bind:id="val"></div>
<div :id="val"></div>  <!--简写-->

  2. v-show 控制显示和隐藏

    每次new Vue扫描带有v-show的元素时，都会动态判断条件。如果条件返回true，就正常显示。如果条件返回false，就自动翻译为display:none，隐藏该元素

    <div id="app">
<div>{{pno}}/{{pcount}} </div>
<button v-show="pno<pcount" @click='ck'> 下一页</button>
</div>
<script src="js/vue.js"></script>
<script>
  var vm = new Vue({
    el:'#app',
    data: { pno: 0, pcount: 3 },
    methods: {
      ck() {
        this.pno++;
      }
    }
  });
</script>

  3. 两个元素，二选一显示: v-if / v-else 在扫描时，如果条件为true，就显示v-if的元素，如条件为false，就显示v-else的元素， v-if和v-else两个元素之间禁止插入其他元素。两个元素必须紧邻。

    1. v-show使用display:none隐藏元素 —— 效率高
    2. v-if v-else是通过删除元素方式隐藏元素 —— 效率低

  4. 多个元素，多选一显示: v-if / v-else-if / v-else 同上显示原理。多了几种可能

  5. 反复生成多个相同结构的元素 v-for :key 一定有一个数组或者数值和 让你可做参照反复生成的元素内容 可以同时获得内容和下标。强烈建议使用 v-for 必须加 :key 需要绑定到要生成的子元素上。

    <li v-for="(item,i) of arr" :key="i">
  <span>{{i}}</span>
  <span>{{item}}</span>
<li>

  6. 绑定函数 v-on /@

    如果既需要传参 还需要传递事件对象e 就需要在绑定时 传递 $event 它是Vue中的关键字 代表Dom的事件对象。

  7. 绑定HTML片段 v-html

    V-htm是一个指令 需要放在开始标签内。

  8. v-cloak指令 Vue加载完成之前 可以临时隐藏个别元素，但是需要自己定义 css 样式 适用于隐藏htm片段

  9. v-text 隐藏内容 但是html片段还在。

  10. v-once 只在加载时 自动绑定一次。之后 变量改变不会在自动刷新页面。

  11. v-pre保持内容中的{{}}不被替换。

  12. v-model双向绑定:

    既能从模型绑定到界面的绑定**(m->v)** 又能将界面的修改 自动更新到模型中**(v->m)**

  1. 单选按钮绑定

  2. 多选按钮单用
    <input type="checkbox" v-model:checked='isAgree'>同意
<button :disabled="!isAgree">提交</button>
  3. 单选列表

    • 选中项改变时，改变的是value属性值，所以 v-model应该写在 select的 value上
    <select v-model:value="city">
  <option value="bj">北京</option>
  <option value="sh">上海</option>
  <option value="nj">南京</option>
  <option value="ly">洛阳</option>
</selecct>

v-model在不同的属性上 绑定的属性也不一样。所以后期使用都可以直接使用 v-model='变量名'

过滤器

专门对原始类型的值进行加工后显示的函数 只要原始属性必须加工才显示就需要使用到过滤器

<div id="app">
  <p>性别：{{sex | sexFilter}}</p>
</div>
<script src="../js/vue.js"></script>
<script>
  Vue.filter('sexFilter',function(value){
    return value=='0' ? '女':'男';
  })
  var vm = new Vue({
    el: "#app",
    data: {
      sex:0,
    }
  });
</script>

==过滤器可以传参， 第一个是实参值，就写在实参列表的第一个位置，调用时形参成会自动传给后一个实参==

多个过滤器可以连用，后一个过滤器收到的是前一个过滤器处理过的值。

<div id="app">
  <p>性别：{{sex|sexFilter('en') |sexIcon}}</p>
</div>
<script src="../js/vue.js"></script>
<script>
  Vue.filter('sexFilter',function(value,lang='cn'){
    if(lang=='cn'){
      return value == '0' ? '女' : '男';
    }else{
      return value == '0' ? 'Female' : 'Male';
    }
    
  })
  Vue.filter('sexIcon',function(value){
    if(value==0 || value=='男' || value=='Male'){
      return 'male.png';
    }else{
      return 'female.png';
    }
  });
  var vm = new Vue({
    el: "#app",
    data: {
      sex:0,
    }
  });
</script>

    组件(Component)

    只要发现一个功能区域需要反复使用，都要先封装为组件。 每个组件内都是一个完整的小社会。

    Vue在扫描真实Dom树的时候，发现组件标签 会在Vue家里找到组件标签同名的组件类型 代替界面中组件标签的位置 创建一个专属的data{}对象 专属专用互不干涉。

    <!-- 引用组件 -->
<my-counter></my-counter> 
<script>
// 创建一个组件 组件名称不适用驼峰 不区分大小写。
Vue.component("my-counter",{
  template:`<span>
    <button @click="change(-1)">-</button>
    <span>{{n}}</span>
    <button @click="change(+1)">+</button>
  </span>`,
  data(){
    return {
      n:0
    }
  },
  methods:{
    change(i){
      this.n+=i;
      this.n<0 && (this.n=0);
    }
  }
})
</script>

    Vue SPA(单页面应用)

    一个应用程序中 只有一个完整的HTML页面，其他所谓的页面 其实都是一些较大的组件而已。所谓的页面跳转实际是更换不同的组件。

    多页面 单页面
    请求次数 每切换一次页面，一定会重新向服务器发送一次请求。——请求次数多 只在首次就把所有组件一次性加载到客户端，之后即使切换"页面”，也只是在客户端本地切换组件而已，无需重复向服务器发送请求——请求次数少
    页面加载效率 每切换一次页面，都要重建整棵DOM树——效率低 只在第一次记载时创建整课DOM树，之后切换"页面”，只是局部更新DOM树的内容。——效率高
    共享资源 每个页面都要重新引用共享的资源文件（jQuery, bootstrap, Qs, axios。。。）,只要切换页面，就要重新从服务器下载共享资源文件。——请求次数多 资源文件只在唯一完整的HTML文件上引用一次。之后每次切换页面，不需要重新从服务器下载共享资源文件。——请求次数少
    页面切换动画 不可能实现页面切换动画。因为两个页面不可能同时存在于浏览器同一个窗口中，所以不可能实现页面过渡动画。 容易实现两个"页面"间切换的过渡动画效果。

    创建一个唯一完整的HTML页面。必须包含vue

Vuex

  1. state保存共享数据
  2. mutations保存函数用于修改共享数据
  3. getters 保存函数用于获取共享数据
  4. action 保存函数异步修改数据 等待数据加载成功后,异步修改数据
//引入
import Vuex from "vuex";

//定义store
const store = new Vuex.Store({
  state: {
    count: 0,
    age: 99
  },
  mutations: {
    increment(state) {
      state.count++;
    },
    clearAge(state){
      state.age=0;
    }
  },
  getters:{
    count(state){
      return state.count;
    }
  },
  actions: {
    modifyAge: context => {
      setTimeout(() => context.commit("clearAge"), 1000);
    } //异步修改
  }
});

new Vue({
  store, //载入vue
  render: h => h(App)
}).$mount("#app")

// .vue
export default {
  computed: {
    count() {
      this.$store.commit("increment"); //可以调用赋值函数
      return this.$store.state.count;
    }
  },
  methods:{
    clearAge(){
      this.$store.dispatch('modifyAge');
    }
  }
};
